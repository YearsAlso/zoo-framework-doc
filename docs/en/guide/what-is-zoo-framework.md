# 简介

## 这个名字很难记
我知道大家想吐槽，为什么要用一个这么拗口的名字，说实话，我开始有想用 zoo 命名，但是让我想起了一系列古早的游戏ID：好名都让xx占了，于是我只能在末尾加上了framework作为后缀以做区分。

## 那么为什么是 zoo
首先，因为我设想把对象关在笼子里，所以自然而然让人想到了动物园。所以是不是听上去有些像依赖注入。之前接触到一系列依赖注入的框架，感觉用起来不错，但是看看python，好像没有那么完善的框架，也可能是我 python 技术栈有限，没有了解到相似的框架。但是依赖注入是真的好用，也可能老法师要疯狂吐槽，但是依赖注入的代表 spring 真的是老少咸宜。
其次，之前是b 站邓园长的粉丝，开始很好奇是什么园长，我开始以为是幼儿园，结果是动物园。。。好吧，那我索性开发个框架就叫动物园吧。以后有机会再重命名

## 不想做 python 的 spring
Spring 固然好用，但是随着开发经验的积累，我
发现，我开发的目的并不再是针对依赖注入，而是希望能以依赖注入为基础，作出一个高效的多线程开发工具，没错，是一个多线程的开发工具，这么想，叫动物园确实不怎么合适，应该叫马戏团。


## 在多线程 zoo 做了哪些工作
首先，项目一如既往的专注于worker slave 模式，提供快速创建worker 的功能，方便用户去延伸自己的开发
